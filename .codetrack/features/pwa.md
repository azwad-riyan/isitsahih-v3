# Feature: PWA (Progressive Web App)
_Created: 2026-06-17 | Last changed: 2026-06-17_

## Purpose
Make IsItSahih installable as a native-like app on Android, iOS, and desktop. Users should see an "Add to Home Screen" / "Install" prompt and be able to use the app offline for the shell.

## Scope
- **Files owned**: `public/sw.js`, `public/manifest.webmanifest`, `public/icon-192.png`, `public/icon-512.png`
- **Files touched**: `index.html`, `index.tsx` (SW registration + install prompt logic)
- **External deps**: None — manual service worker (no vite-plugin-pwa)

## Current State
Fully implemented. Install prompt is live (Chrome/Edge/Android). iOS uses apple-touch-icon + apple-mobile-web-app-capable. Service worker caches app shell for offline. Analytics track install events via `track()` → Netlify function.

**Known gap**: `icon-192.png` and `icon-512.png` must be placed in `public/` by the developer — they are NOT auto-generated by the build.

## Change Log

### [2026-06-17] Make PWA fully installable — add PNG icons + improve manifest
**Why**: Chrome/Android requires actual PNG icons at 192×192 and 512×512 to trigger the install prompt. SVG alone does not satisfy Chrome's installability criteria. Also manifest lacked `maskable` purpose needed for Android adaptive icons and `display_override` for better multi-browser support.
**What**: + `icon-192.png` and `icon-512.png` references in manifest; + separate `maskable` purpose entries; + `display_override: ["standalone","minimal-ui"]`; + `shortcuts` entry; ~ `apple-touch-icon` in `index.html` now points to PNG; ~ SW cache bumped to `isitsahih-v2`, PNG icons added to precache list; + `.codetrack/` initialized
**How**: Icons listed twice in manifest (once with `purpose: any`, once with `purpose: maskable`) — this is the correct pattern per the spec; a single entry with `"any maskable"` space-separated string also works but separate entries are clearer.
**Impact**: ~ `public/manifest.webmanifest`, ~ `public/sw.js`, ~ `index.html`; ! Developer must place `icon-192.png` + `icon-512.png` in `public/` manually
**Ref**: User request — make installable PWA
